# GCS_Semana2_Baseline_MoralesChristian

## Objetivo del proyecto
Este proyecto tiene como objetivo practicar la gestión de la configuración del software (GCS), aplicando buenas prácticas de versionado, creación de líneas base, control de cambios y trazabilidad utilizando herramientas como Git.

## Estructura del repositorio
- `/docs`: Contiene la documentación del proyecto.
- `/src`: Contiene el código fuente.
- `/tests`: Contiene los casos de prueba.
- `/config`: Archivos de configuración.
- `/scripts`: Scripts de despliegue y otros.
